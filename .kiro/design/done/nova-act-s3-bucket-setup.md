## Current S3 Architecture

### Existing Buckets (from configuration.json)

Your `enabledRegions` configuration creates buckets in:
- **eu-central-1** (default/primary): `<account>-nova-act-qa-studio-modular-artefacts-eu-central-1`
- **us-east-1**: `<account>-nova-act-qa-studio-modular-artefacts-us-east-1` ✅ **Used by Nova Act**
- **us-west-2**: `<account>-nova-act-qa-studio-modular-artefacts-us-west-2`
- **ap-southeast-2**: `<account>-nova-act-qa-studio-modular-artefacts-ap-southeast-2`

### Cross-Region Replication

All regional buckets replicate TO the eu-central-1 (default) bucket:
```
us-east-1 ──────┐
us-west-2 ──────┼──> eu-central-1 (primary)
ap-southeast-2 ─┘
```

**Configuration:**
- Source: Regional buckets (versioned)
- Destination: eu-central-1 bucket
- Replication: Objects, delete markers, tags
- Role: `nova-act-qa-studio-modular-replication-role-<region>`

## Nova Act GA Requirements

### Workflow Definition Storage

Nova Act requires an S3 bucket in **us-east-1** for:
1. **Workflow definitions** - Metadata about your workflows
2. **Execution traces** - Step-by-step execution logs
3. **Session artifacts** - Screenshots, browser state (optional)

### Bucket Permissions Required

Nova Act service needs:
- `s3:GetObject` - Read workflow definitions
- `s3:PutObject` - Write execution traces
- `s3:ListBucket` - List workflow artifacts

**Implementation:** Add bucket policy or use IAM role permissions (handled by SDK)

## Integration Points

### 1. Workflow Definition Creation

When creating a workflow definition, specify the us-east-1 bucket:

```bash
# AWS CLI
aws nova-act create-workflow-definition \
  --name "qa-studio-wizard" \
  --description "QA Studio wizard mode workflow" \
  --s3-bucket-name "<account>-nova-act-qa-studio-modular-artefacts-us-east-1" \
  --region us-east-1

# Nova Act CLI
act workflow create \
  --name "qa-studio-wizard" \
  --region us-east-1 \
  --s3-bucket-name "<account>-nova-act-qa-studio-modular-artefacts-us-east-1"
```

### 2. Worker Environment Variables

Add to ECS task definition:

```typescript
// lib/worker-stack.ts
container.addEnvironment(
  'NOVA_ACT_S3_BUCKET_US_EAST_1',
  `${this.account}-${this.baseName}-artefacts-us-east-1`
);
```

### 3. IAM Permissions

Worker task role already has S3 permissions via:
```typescript
this.artefactsBucket.grantReadWrite(this.taskDefinition.taskRole);
```

But you need to extend this to include the us-east-1 bucket:

```typescript
// Add to worker-stack.ts
const usEast1BucketArn = `arn:aws:s3:::${this.account}-${this.baseName}-artefacts-us-east-1`;

taskDefinition.taskRole.addToPolicy(new PolicyStatement({
  effect: Effect.ALLOW,
  actions: [
    's3:GetObject',
    's3:PutObject',
    's3:ListBucket',
    's3:DeleteObject'
  ],
  resources: [
    usEast1BucketArn,
    `${usEast1BucketArn}/*`
  ]
}));
```

## Data Flow

### Current Flow (Preview API)
```
Worker (eu-central-1)
  │
  ├─> Nova Act Preview API (region-agnostic)
  │
  └─> S3 eu-central-1 (artifacts)
```

### New Flow (GA Service)
```
Worker (eu-central-1)
  │
  ├─> Nova Act GA Service (us-east-1)
  │     │
  │     └─> S3 us-east-1 (workflow definitions, traces)
  │           │
  │           └─> Replication ──> S3 eu-central-1
  │
  └─> S3 eu-central-1 (browser recordings, screenshots)
```

**Key Points:**
- Worker stores browser artifacts directly to eu-central-1 (no change)
- Nova Act stores execution traces to us-east-1
- Replication brings us-east-1 data back to eu-central-1
- All data eventually consolidated in eu-central-1

## Artifact Storage Strategy

### What Goes Where

| Artifact Type | Storage Location | Reason |
|---------------|------------------|--------|
| Browser recordings (DCV) | eu-central-1 | Generated by worker, low latency |
| Screenshots (NovaAct SDK) | eu-central-1 | Generated by worker, low latency |
| Worker logs | eu-central-1 | CloudWatch in worker region |
| Workflow definitions | us-east-1 | Required by Nova Act service |
| Execution traces | us-east-1 | Generated by Nova Act service |
| Step metadata | us-east-1 | Generated by Nova Act service |

### Replication Benefits

1. **Data Residency:** All data eventually in eu-central-1
2. **Compliance:** Meets data locality requirements
3. **Backup:** Automatic cross-region backup
4. **Access:** Can access all artifacts from primary region

## Cost Implications

### Storage Costs

**No additional storage costs** - using existing buckets

### Replication Costs

**Existing:** Already paying for replication from regional buckets to eu-central-1

**New:** Nova Act artifacts in us-east-1 will replicate to eu-central-1
- Estimated size: ~1-5MB per workflow run (traces, metadata)
- 1000 runs/day = 5GB/day = 150GB/month
- Replication cost: $0.02/GB = $3/month

**Total incremental cost:** ~$3-5/month for replication

### Data Transfer Costs

- Worker → Nova Act (us-east-1): ~$0.02/GB (cross-region)
- Nova Act → S3 (us-east-1): Free (same region)
- S3 replication (us-east-1 → eu-central-1): $0.02/GB

## Troubleshooting

### Issue: "Access Denied" when creating workflow definition

**Cause:** Worker task role doesn't have S3 permissions for us-east-1 bucket

**Solution:**
```typescript
// Add to worker-stack.ts
const regionalBucketArns = this.regionalBucketArns; // Already collected

taskDefinition.taskRole.addToPolicy(new PolicyStatement({
  effect: Effect.ALLOW,
  actions: ['s3:*'],
  resources: [
    ...regionalBucketArns,
    ...regionalBucketArns.map(arn => `${arn}/*`)
  ]
}));
```

### Issue: "Bucket not found" error

**Cause:** Bucket name incorrect or not created

**Solution:**
1. Check configuration.json has `us-east-1` in `enabledRegions`
2. Redeploy worker stack to create bucket
3. Verify bucket name format: `<account>-<baseName>-artefacts-us-east-1`

### Issue: Replication not working

**Cause:** Replication role missing permissions

**Solution:**
Check replication role has:
- Read permissions on source bucket (us-east-1)
- Write permissions on destination bucket (eu-central-1)
- Trust relationship with s3.amazonaws.com
